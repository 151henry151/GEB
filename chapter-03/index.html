Chapter III: Figure and Ground

In some pictures you can look at the “thing” in the middle—that’s the figure. Or you can look at the “stuff around it”—that’s the ground. Same picture, two ways to see it.

In the game of theorems, the “figure” is the list of strings we can make with the rules. The “ground” is everything else. Sometimes the figure is easy to list, but the ground is hard to pin down—you can’t always tell in a simple way what’s “not a theorem.” So the ground can hide information the figure doesn’t show.

Figure and ground in art

In art, “figure” is what we focus on (e.g. a vase); “ground” is the rest (e.g. the background, or the space around the vase). Some images flip: you can see either two faces or a vase. The same marks on the page support two different “readings.”

Figure and ground in formal systems

In a formal system, the “figure” is the set of theorems (strings we can derive). The “ground” is the set of non-theorems. The figure is “recursively enumerable”: we can list all theorems one by one. But can we list all non-theorems? Sometimes we can (then the set is “recursive”); sometimes we can’t. So the ground doesn’t always have the same kind of structure as the figure.

Why it matters

That difference—between “listable” theorems and “maybe not listable” non-theorems—will matter for Gödel. It’s like asking: does the figure contain all the information that the ground has? Not necessarily.

Figure and ground in perception and in logic

Hofstadter compares the figure–ground distinction in visual perception (and in Escher’s art) to the distinction between theorems and non-theorems in a formal system. In both cases we have a “positive” set (what we see as figure; what we can derive) and a “negative” set (ground; non-theorems). The question is whether the two sets carry the same information—whether the ground is determined by the figure in a computable way.

Recursively enumerable vs recursive sets

The set of theorems of a formal system is recursively enumerable: there is a procedure that lists exactly the theorems (e.g. enumerate all proofs). The set of non-theorems is the complement. If there is a procedure that decides, for any string, whether it is a theorem, then both the set of theorems and the set of non-theorems are “recursive” (decidable). But if there is no decision procedure for theoremhood, then the set of non-theorems is not recursively enumerable in the same simple way. So “figure” (theorems) and “ground” (non-theorems) can have different computational structure.

Setting up incompleteness

This prepares the reader for the idea that in rich systems like TNT, the “ground” (true but unprovable sentences) may not be listable by the system’s own means—so the figure (theorems) does not fully determine the ground.
