Chapter XVII: Church, Turing, Tarski, and Others

Some programs stop; some run forever. Can we always tell which? No. There’s no perfect recipe that, for every program, says “it will stop” or “it won’t.” So some questions about programs have no answer we can get by one fixed rule. Logic and computers both have limits.

Church and Turing

Church and Turing showed that some problems are undecidable: there is no algorithm that always gives the right yes/no answer. The halting problem is the most famous: given a program and an input, does the program eventually stop? No general algorithm can decide that for every program and input.

Tarski

Tarski showed that truth (in a rich enough language) cannot be defined inside the same language. So "this sentence is true" cannot be consistently defined in the object language. That connects to the liar paradox and to Gödel.

Why it matters

Undecidability, uncomputability, and the indefinability of truth are three limits: on what we can compute, what we can decide, and what we can say about truth from inside a system.

Church, Turing, and the halting problem

Hofstadter discusses Church's thesis (the identification of "computable" with lambda-definable / recursive) and Turing's characterization of computation by machines. The halting problem is undecidable: there is no Turing machine that, given a description of a machine M and an input x, always correctly determines whether M halts on x. The proof is by diagonalization: assume such a machine H exists, then build a machine that does the opposite of what H says, leading to contradiction.

Tarski and the indefinability of truth

Tarski showed that a sufficiently rich formal language cannot contain its own truth predicate: if we could define "true" in the language, we could construct a liar sentence. So truth (for that language) is definable only in a richer meta-language. This parallels Gödel's incompleteness: provability can be expressed inside the system, but truth (for all sentences) cannot.

Connections

Gödel's theorem, the undecidability of the halting problem, and Tarski's indefinability result are all linked by diagonalization and self-reference. They mark fundamental limits on formal systems and computation.

2. The diagonalization family

Cantor, Gödel, Turing, and Tarski all built an object that disagrees with its own row in a complete table. Same spine, different costumes.

Cantor (1891)
Oracle assumption
All reals can be listed in one sequence.
Diagonal move
Build a digit that differs from entry n in the n-th decimal place.
Contradiction
That number is not on the list — yet the list was complete.
Result
No complete list of real numbers exists.
Gödel (1931)
Oracle assumption
A formal system proves every arithmetic truth.
Diagonal move
Sentence G: “I am not provable” (fixed point / arithmoquining).
Contradiction
If provable → falsehood; if not provable → true but unprovable.
Result
No complete consistent formal system for arithmetic.
Turing (1936)
Oracle assumption
Program H decides halting for every program and input.
Diagonal move
Program D does the opposite of H on diagonal inputs.
Contradiction
H cannot describe D correctly on D’s own index.
Result
No general halting algorithm exists.
Tarski (1933)
Oracle assumption
Truth predicate True(x) definable in the language.
Diagonal move
“True(⌈this sentence⌉) is false” — liar, formalized.
Contradiction
The sentence is true iff it is false.
Result
Truth for a language cannot be defined inside that language.
Unifying template
  1. Assume a complete self-describing mechanism M exists.
  2. Use M to construct D that “disagrees with” M’s answers (diagonal).
  3. Ask: what does M say about D?
  4. Both answers lead to contradiction.
  5. Therefore M cannot exist.

Cantor, Gödel, Turing, and Tarski all proved the same theorem. The subject changed — sets, formulas, programs, truth — but the proof never did.

3. Church’s thesis

Any function that can be computed by an effective procedure can be computed by a Turing machine.

Why it’s a thesis, not a theorem. Church’s thesis cannot be proved — because “effective procedure” is informal. We cannot formally prove that our formal definition of computation captures everything we intuitively mean by computation.

What we can prove: many formal models — Turing machines, lambda calculus, recursive functions, register machines, ordinary programming languages — all compute the same class of functions. They are equivalent to each other.

Every reasonable model of computation turns out to be equivalent. This is evidence — though not proof — that Church’s thesis is correct.

If Church’s thesis is true, then Turing’s result is not just about Turing machines — it is about all computation, including human computation. The halting problem is undecidable not only for Turing machines but for any computational process whatsoever — including, if the thesis holds, the human brain.

Is human reasoning subject to these limits? Or does jumping out of the system — reasoning about formal systems from outside — escape them?

Three mathematicians. Three problems. Three domains — sets, programs, truth. One proof. The diagonal argument is the most powerful tool in the foundations of mathematics. It marks the edge of what any formal system, any computer, any language can do from inside itself.
